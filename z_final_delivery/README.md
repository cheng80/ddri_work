# z_final_delivery

이 폴더의 최종 목표는 `works/`를 참고하지 않아도 단독으로 운영 가능한 최종 전달 패키지를 만드는 것이다.

즉 이 경로는 단순 요약 폴더가 아니라, 기존 실험 결과를 정리해 필요한 입력 데이터, 최종 출력 데이터, 핵심 근거 문서까지 이 안으로 다시 가져와 `단일 정본 노트북`과 `최종 서비스 문서`로 묶는 경로다.

현재 `works/`는 여전히 근거 저장소이자 추가 실험 경로로 유지한다. 하지만 `z_final_delivery`는 최종적으로 이 폴더만 분리해도 읽고 실행할 수 있는 자기완결형 구조를 목표로 한다.

## 구성

2. [02_web_service_final](/Users/cheng80/Desktop/ddri_work/z_final_delivery/02_web_service_final)

## 작업 원칙

- 새 실험은 계속 `works/`에서 수행한다
- 확정된 결과만 이 폴더로 다시 가져와 정리한다
- 입력 데이터는 이 폴더 아래 `input` 성격 경로로 복사해 둔다
- 출력 데이터도 이 폴더 아래 `output` 성격 경로에서만 관리한다
- 필요한 근거 문서는 최소한으로 다시 정리해 `doc` 성격 경로에 둔다
- 분석/ML 축은 가능한 한 `단일 정본 노트북 1개`를 중심으로 정리한다
- 웹/서비스 축은 API, 후처리, 화면 연결 정의만 따로 묶는다
- `works` 마무리 전까지 `master plan`, `TODO`, `cleanup`은 계속 유효하다

## 현재 기준

- `02_web_service_final`

## 최종 상태 목표

- `z_final_delivery`만 별도로 분리해도 구조가 유지된다
- 노트북은 이 폴더 안의 입력 데이터만 읽는다
- 결과 표와 서비스 연결용 산출물도 이 폴더 안에만 쓴다
- 필요한 설명 문서와 근거 문서도 이 폴더 안에서 닫힌다
