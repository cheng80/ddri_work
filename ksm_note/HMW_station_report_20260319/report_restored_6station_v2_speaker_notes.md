# 강남구 따릉이 6개 대여소 예측 분석 V2 스피치 노트

## 1페이지. 표지 / 분석 목적

안녕하세요. 이번 발표는 강남구 따릉이 대여소 중 랜덤하게 선정한 6개 대여소를 대상으로 수요 예측 과정을 점검한 결과입니다.

이번 발표의 핵심은 군집화 결과를 보여드리는 것이 아니라, 제한된 수의 대여소만으로도 예측 파이프라인이 안정적으로 작동하는지 확인하는 데 있습니다.

즉, 지금 단계는 전체 운영 모형을 완성하는 단계라기보다, 예측 구조가 해석 가능하게 작동하는지 먼저 보는 파일럿 단계라고 이해해주시면 됩니다.

## 2페이지. 이번 단계에서 군집화를 하지 않은 이유

먼저 왜 군집화를 하지 않았는지 말씀드리겠습니다.

이번 분석 범위는 강남구 전체 대여소가 아니라 랜덤 선정한 6개 대여소입니다. 이 정도 규모에서는 굳이 군집화를 먼저 적용하지 않아도 개별 대여소의 패턴과 예측 성능을 직접 확인할 수 있습니다.

오히려 표본 수가 작은 상태에서 군집화를 전면에 두면, 군집 결과 자체를 일반화해서 해석하기가 더 어려워질 수 있습니다.

그래서 이번 단계에서는 군집화 없이 예측 구조와 성능 해석이 가능한지 먼저 점검하는 쪽으로 방향을 잡았습니다.

## 3페이지. 향후 전체 확장 시 군집화가 필요한 이유

다만 군집화가 필요 없다는 뜻은 아닙니다.

향후 강남구 전체 대여소로 범위를 넓히게 되면, 대여소마다 입지 조건과 이용 패턴이 굉장히 다르기 때문에 이를 하나의 구조로만 해석하기는 어렵습니다.

그때는 대여소를 station-level에서 군집화해서 공간 역할을 먼저 정리하고, 그 위에서 예측 성능 차이를 해석하는 방식이 더 적절합니다.

즉, 현재는 군집화를 생략했지만, 전체 확장 단계에서는 다시 필요한 분석 축이라고 보시면 됩니다.

## 4페이지. 군집화는 현재 결과가 아니라 향후 참고 프레임

이 페이지의 시각화는 이번 6개 대여소 예측 결과 자체가 아니라, 향후 전체 대여소 확장 분석에서 참고할 수 있는 프레임입니다.

산점도는 군집이 어느 정도 분리되는지를 보여주고, 히트맵은 군집별로 어떤 특성이 다른지를 요약합니다.

따라서 여기서는 군집 결과를 확정적으로 제시한다기보다, 나중에 전체 대여소를 다룰 때 어떤 방식으로 구조를 해석할 수 있는지를 보여주는 참고 화면으로 이해하시면 됩니다.

## 5페이지. 전체 예측 파이프라인 안에서 현재 단계의 위치

현재 단계는 6개 대여소를 대상으로 예측 파이프라인을 점검하는 단계입니다.

구체적으로는 전처리, 패턴 기반 feature 생성, Ridge 예측, 오차 해석까지 한 번에 연결해보는 것이 목표였습니다.

이후 전체 확장 단계에서는 강남구 전체 대여소를 수집하고, 그 위에서 군집화를 다시 도입해 대여소 유형별 예측 성능까지 함께 해석하는 방향으로 갈 수 있습니다.

## 6페이지. 데이터와 전처리 구조

다음은 데이터와 전처리 구조입니다.

관측 기간은 2023년부터 2025년까지이고, train, valid, test를 연도 기준으로 나눴습니다.

기본적인 품질 점검 결과를 보면 결측, 음수, 중복 시간행은 거의 없었고 시계열 길이도 일정했습니다.

즉, 이번 분석에서는 데이터 정제 자체보다 예측 구조를 설계하고 해석하는 데 집중할 수 있는 상태였습니다.

## 7페이지. 분할 구조와 기본 패턴

여기서는 데이터 분할과 기본 패턴을 보여드립니다.

train은 2023년, valid는 2024년, test는 2025년으로 두었고, day_type은 weekday와 offday로 나눴습니다.

그래프를 보시면 weekday가 offday보다 평균 대여와 반납이 더 높고, 출근과 퇴근 시간대에 피크가 뚜렷하게 나타납니다.

이 기본 패턴이 뒤에서 설명드릴 패턴 기반 feature 설계의 출발점입니다.

## 8페이지. 패턴 기반 feature + Ridge 구조

이번 분석에서는 원천 count를 그대로 넣기보다는, 먼저 시간 패턴을 구조화해서 feature로 만드는 방식을 사용했습니다.

기본적으로 day_type별 시간 패턴을 만들고, 여기에 month, year, hour 보정값을 얹어서 pattern_prior와 corrected_pattern_prior 같은 feature를 구성했습니다.

모델은 Ridge 회귀를 사용했는데, 이유는 최고 성능의 블랙박스 모델보다는 station별 차이를 비교 가능하게 설명하는 기준 모델이 더 중요했기 때문입니다.

## 9페이지. 가중치 구조는 어떤 패턴을 반영하는가

이 페이지는 월, 연도, 시간 가중치가 무엇을 반영하는지를 보여줍니다.

month_weight는 계절성, year_weight는 연도별 수준 차이, hour_weight는 세부 시간대 편차를 보정합니다.

특히 hour_weight는 출퇴근 피크처럼 반복적으로 발생하는 시간대 오차를 잡는 데 중요한 역할을 합니다.

반면 year_weight는 새로운 미래 연도를 미리 정확히 예측하는 값이라기보다는, 해당 연도의 수준 차이를 반영하는 사후적 보정값에 가깝습니다.

## 10페이지. Ridge 튜닝과 전체 성능 해석

여기서는 Ridge alpha 선택과 전체 성능을 요약합니다.

target별로 alpha가 다르게 선택됐고, 일부 station은 아주 작은 alpha를, 일부는 큰 alpha를 사용했습니다. 이건 station별 패턴 복잡도가 다르다는 의미로 볼 수 있습니다.

전체 성능을 보면 return_count가 rental_count보다 평균 R^2가 약간 더 높았고, train-valid-test 간 격차도 과도하게 크지 않았습니다.

즉, 기준 모델로서는 비교적 안정적으로 작동했다고 볼 수 있습니다.

## 11페이지. 파일럿에서 주목한 6개 station

이번 파일럿에서 상대적으로 주목한 6개 station은 포스코사거리, 매봉역 3번출구앞, 수서역 5번출구, 자곡사거리, 압구정역 2번 출구 옆, 수서역 1번출구 앞입니다.

지도와 성능 그래프를 보시면, 같은 모델 구조를 사용해도 station별 재현력 차이는 분명하게 존재합니다.

이 지점이 중요한 이유는, 이후 전체 대여소로 확장할 때 대여소 차이를 설명할 체계가 필요하다는 점을 보여주기 때문입니다.

## 12페이지. 모델은 무엇을 가장 크게 참고했는가

모델 중요도를 보면 거의 모든 station에서 corrected_pattern_prior가 가장 큰 비중을 차지합니다.

그 다음으로는 pattern_prior와 hour_weight가 뒤를 잇습니다.

즉, 이 모델의 중심은 패턴 기반 사전 예측값을 얼마나 잘 만들고, 거기에 시간대 보정을 얼마나 잘 얹느냐라고 정리할 수 있습니다.

## 13페이지. 어디서 오차가 커졌는가

오차 hotspot은 주로 17시에서 19시 사이, 그리고 4월에서 6월, 9월에서 10월 사이 구간에 집중됐습니다.

station별로는 2348, 2377, 2335에서 상대적으로 큰 오차가 관찰됐습니다.

이 결과는 현재 구조만으로는 저녁 피크나 계절 전환 구간의 수요 변동을 완전히 설명하기 어렵다는 뜻이고, 이후에는 날씨, 이벤트, 재배치 같은 외생 변수 보강이 필요하다는 근거가 됩니다.

## 14페이지. 결론과 다음 단계

마지막으로 정리드리겠습니다.

현재 분석 범위는 6개 대여소 파일럿입니다. 이번 단계에서는 군집화를 생략하고 예측 구조 자체가 작동하는지 확인하는 데 집중했습니다.

예측 구조는 패턴 기반 feature와 Ridge 회귀 조합으로 충분한 해석 가능성을 보여줬고, 동시에 station별 재현력 차이와 시간대 hotspot을 통해 현재 구조의 한계도 드러났습니다.

다음 단계에서는 강남구 전체 대여소로 범위를 넓히고, 그때는 station-level 군집화를 다시 도입해서 대여소 유형별로 예측 결과를 해석하는 방향으로 확장할 수 있습니다.
