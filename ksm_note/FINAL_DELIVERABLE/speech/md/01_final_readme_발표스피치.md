# 발표 스피치 01: 외곽 Top3 최종 모델 패키지 (`final/README.md`)

## 1. 오프닝 (약 20초)
안녕하세요.  
지금부터 외곽 주거형 Top3 대여소에 대해, 최종 모델 패키지 결과를 공유드리겠습니다.  
핵심은 시간대별 수요 예측 성능을 확인하고, 실제 운영 의사결정에 쓸 수 있는 수준인지 점검하는 것입니다.

## 2. 모델/데이터/평가 기준 (약 30초)
이번 결과는 LightGBM 회귀 모델을 사용했고, feature 전략은 `baseline + is_weekend`입니다.  
시간대, 요일, 월은 원핫 인코딩했고, 2025 데이터를 최종 예측 대상으로 평가했습니다.  
평가는 RMSE, MAE, R2, 그리고 정수 반올림 정확일치 기준으로 확인했습니다.

## 3. 클래스 의미 설명 (약 20초)
혼동행렬의 `0, 1, 2, 3, 4`는 클래스 번호가 아니라 실제 시간당 대여건수 자체입니다.  
즉 0은 0건, 1은 1건, 2는 2건을 의미합니다.  
기존 `3+` 버킷은 3건 이상을 하나로 묶은 단순화 지표입니다.

## 4. 대여소별 성능 핵심 (약 50초)
대여소별로 보면 2392는 대여량이 희소해서 정확일치율이 매우 높게 나왔고,  
2359와 3643은 상대적으로 수요 변동이 커서 난이도가 높은 패턴입니다.

- 2359: 정확일치 약 52.8%
- 2392: 정확일치 약 92.7%
- 3643: 정확일치 약 58.3%

해석 포인트는, 동일 모델이라도 대여소 특성에 따라 예측 난이도가 크게 다르다는 점입니다.

## 5. 전체 Train/Valid/Test 점수 (약 50초)
Top3 통합 기준으로 보면:

- Train(2023): RMSE 0.5425, R2 0.7120
- Valid(2024): RMSE 0.8098, R2 0.2938
- Test(2025): RMSE 0.7282, R2 0.3571

훈련 성능 대비 검증/테스트가 내려가지만, 검증과 테스트 사이가 비정상적으로 벌어지지는 않았습니다.  
즉 일반화 성능은 제한적이지만 완전히 무너진 형태는 아니며, 운영 실험 단계에서는 사용 가능한 수준으로 판단했습니다.

## 6. 혼동행렬 해석 (약 45초)
통합 혼동행렬을 보면, 실제 0~1건 구간은 비교적 잘 맞추고,  
고수요 구간(특히 4건 이상)은 예측이 보수적으로 나오는 경향이 있습니다.  
즉 모델이 급격한 피크를 과소추정하는 경향이 있어, 피크 시간대 운영에서는 버퍼 정책이 필요합니다.

## 7. 결론 및 운영 시사점 (약 40초)
정리하면:

1. 현재 모델은 시간대별 수요 방향을 파악하는 데 유효하다.
2. 하지만 고수요 피크 대응에는 추가 보정이 필요하다.
3. 따라서 예측값 단독 운영보다는 안전버퍼를 포함한 정책형 의사결정이 적합하다.
