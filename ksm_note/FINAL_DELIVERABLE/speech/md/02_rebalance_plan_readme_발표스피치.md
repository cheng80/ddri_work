# 발표 스피치 02: 재배치 계획 (`rebalance_plan/README.md`)

## 1. 오프닝 (약 20초)
이제 두 번째로, 예측 결과를 실제 운영행동으로 연결한 재배치 계획을 설명드리겠습니다.  
목표는 여유 대여소에서 부족 대여소로, 시간대별 이동 권고를 만드는 것입니다.

## 2. 입력과 기본 가정 (약 35초)
입력은 final 단계에서 만든 2025 예측 결과를 그대로 사용했습니다.  
그리고 재배치 계산을 위해 다음 가정을 둡니다.

- 필요대수(예측 기준) = 예측 대여건수 + 안전버퍼 1대
- 대여소별 기준재고는 연간 평균 필요대수의 반올림값
- 시간대별로 기준재고 대비 부족/여유를 계산

여유가 큰 대여소에서 부족이 큰 대여소로 먼저 보내는 greedy 방식으로 이동권고를 만들었습니다.

## 3. 기준재고 결과 (약 25초)
계산된 baseline 재고는 다음과 같습니다.

- 2359: 2대
- 2392: 1대
- 3643: 2대

이 값은 고정 운영재고의 초깃값으로 해석할 수 있습니다.

## 4. 재배치 결과 요약 (약 45초)
전체적으로 2025년 8,760시간에 대해:

- 이동 권고 레코드 수: 57건
- 총 이동 권고 대수: 57대
- 예측 기준으로 부족 발생 시간(어느 한 대여소라도): 1,280시간

즉, 현재 단순 규칙에서도 특정 시간대에는 분명히 재배치가 필요하다는 신호가 확인됩니다.

## 5. 예측기준 vs 실제기준 차이 (약 50초)
중요한 점은 실제 부족시간이 예측 부족시간보다 더 크게 나타났다는 점입니다.  
이건 모델 오차와 단순 재고가정 때문에 발생합니다.

- 예측 기준 부족시간: 1,280시간
- 실제 기준 부족시간: 2,690시간

해석하면, 지금 결과는 재배치의 방향성은 제공하지만  
실운영 적용 전에는 실시간 재고와 이동 제약을 넣은 보정이 반드시 필요합니다.

## 6. 현재 계획의 의미와 한계 (약 45초)
현재 계획의 의미:

1. 시간대별 “누가 부족하고 누가 여유인지”를 구조적으로 계산했다.
2. 이동 권고를 데이터 테이블로 바로 뽑아냈다.

현재 한계:

1. 실시간 재고 미반영
2. 차량/인력 제약 미반영
3. 이동 거리/시간/비용 최적화 미반영

즉 의사결정 프로토타입으로는 유효하지만, 운영 최적화 단계는 아직 남아 있습니다.

## 7. 다음 단계 제안 (약 35초)
다음 단계는 세 가지입니다.

1. 실시간 재고 API 또는 재고 스냅샷 결합
2. 차량 수·기사 수·작업시간 제약 반영
3. 비용 최소화 기준의 수송 최적화(예: min-cost flow) 적용

이 단계까지 가면 “예측”에서 “실행 가능한 재배치 운영”으로 넘어갈 수 있습니다.

## 8. 마무리 멘트 (약 15초)
정리하면, 이번 결과는 재배치 운영의 출발선으로 충분합니다.  
다음 스프린트에서 실시간 재고와 최적화 모델을 결합해 운영 의사결정 정확도를 높이겠습니다.
