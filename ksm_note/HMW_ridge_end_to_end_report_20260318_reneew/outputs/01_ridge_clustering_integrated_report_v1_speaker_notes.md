## 페이지 2. 군집화 분석 개요

### 실제 스피치

“먼저 왜 군집화가 먼저였는지 설명드리겠습니다.  
대여량 규모만으로는 각 대여소가 업무 중심인지, 주거 중심인지, 생활·상권 혼합인지 설명하기 어렵습니다.  
그래서 반납 시간대 비율, 순유입, 교통 접근성을 이용해 대여소의 공간 역할을 먼저 분류했습니다.”

### 유의점

- 군집화는 예측 전의 구조 파악 단계라는 점을 강조한다.
- 예측으로 바로 넘어가는 연결 역할만 한다.

## 페이지 3. 군집화 분석에서 사용한 기준

### 실제 스피치

“군집화 타겟은 station-level입니다.  
즉 하루 수요를 맞추는 것이 아니라, 각 대여소가 어떤 공간 역할을 갖는지 먼저 분류하는 작업입니다.  
핵심 피처는 오전·점심·저녁 반납 비율, 아침·저녁 순유입, 지하철 접근성, 버스정류장 밀도입니다.”

### 유의점

- 7개를 다 읽더라도 의미 위주로 짧게 읽는다.
- 변수명은 보조 설명으로만 쓴다.

## 페이지 4. 데이터 기준과 전처리 요약

### 실제 스피치

“군집화 분석은 2023년부터 2025년까지 공통 운영 대여소를 기준으로 했고, 전체 로그 약 289만 건 중 약 266만 건을 최종 사용했습니다.  
핵심은 결측치보다 비정상 이용시간, 즉시 반납, 공통 기준 밖 대여소 데이터를 정리해 공간 역할을 안정적으로 읽을 수 있는 로그를 확보한 점입니다.”

### 유의점

- 세부 제외 사유를 길게 읽지 않는다.
- `분석 가능한 로그 확보`라는 의미만 남긴다.

## 페이지 5-6. 군집 분리와 프로파일 요약

### 실제 스피치

“이 부분은 군집이 실제로 분리되는지와, 무엇이 다른지를 보여주는 페이지입니다.  
PCA 산점도는 군집이 완전히 겹치지 않고 역할별로 분리되는 경향을 보여주고, 히트맵은 각 군집에서 어떤 시간대 반납 비율과 순유입, 교통 접근성이 상대적으로 높은지 요약해 줍니다.  
즉 PCA는 분리 정도, 히트맵은 차이의 내용이라고 보시면 됩니다.  
여기서 중요한 해석은 군집이 단순히 대여량 규모 차이로만 갈린 것이 아니라, 출퇴근 시간대 흐름과 접근성 조합이 다른 역할 집단으로 나뉜다는 점입니다.”

### 유의점

- PCA와 히트맵의 역할을 나눠 설명한다.
- 셀 하나하나를 읽지 않는다.
- PCA는 `완전 분리`를 주장하는 그림이 아니라 `역할별 분리 경향`을 보여주는 근거로 설명한다.
- 히트맵은 `어느 군집이 높다`보다 `군집마다 패턴 조합이 다르다`는 해석에 쓴다.

## 페이지 7-8. 군집 대표 대여소와 공간 분포

### 실제 스피치

“이 페이지는 군집이 실제 공간 분포와 어떻게 연결되는지를 보여줍니다.  
정적 지도에서 강남구 내 군집 위치를 먼저 보고, 아래 대표 대여소 예시로 군집 해석을 더 직관적으로 이해할 수 있습니다.  
예를 들어 수서역이나 포스코사거리 같은 지점은 아침 도착 업무 집중형 해석과 잘 맞고, 청담역이나 세곡동 쪽은 주거 도착형 또는 외곽 주거형 해석과 연결됩니다.  
즉 군집 라벨이 추상적인 번호가 아니라, 실제 강남구 생활권 구조와 맞물린 공간 역할이라는 점을 이 페이지에서 확인할 수 있습니다.”

### 유의점

- 대표 대여소는 2~3개만 읽는다.
- 지도에서는 공간 분포만 짚고 넘어간다.
- 지도 해석은 `어디에 몰려 있는가` 수준까지만 하고, 개별 좌표 설명으로 들어가지 않는다.

## 페이지 9. 예측 분석 문제 정의

### 실제 스피치

“군집화는 역할을 설명하지만, 특정 시점의 수요 변화를 직접 예측하지는 못합니다.  
운영에서는 어느 시간대에 어느 대여소의 자전거 수요 변화가 커질지를 알아야 하기 때문에, 다음 단계로 예측 분석이 필요합니다.  
이번 예측 타깃은 자전거 수요 변화량이고, 단위는 station-hour입니다.”

### 유의점

- 운영 활용 관점으로 넘어간다.
- `station-hour`를 한 번만 짚고 지나간다.

## 페이지 10-12. Ridge 회귀 분석 개요와 데이터 구성

### 실제 스피치

“예측 단계의 목표는 자전거 수요 변화량을 설명 가능한 방식으로 예측하는 것입니다.  
흐름은 데이터 정제, 공선성 정리, 누수 점검, 월별 반복 패턴 진단, sample weight 적용, Ridge 학습과 성능 검증입니다.  
학습은 2023~2024, 테스트는 2025 기준이고, 최종 입력 변수는 16개입니다.  
컬럼 구성은 시간, 기상, 과거 이력, 군집, 그리고 학습 가중치로 이해하시면 됩니다.”

### 유의점

- 컬럼 표는 전부 읽지 않는다.
- `시간 + 기상 + 과거 이력 + 군집` 묶음으로 설명한다.

## 페이지 13-15. 데이터 정제와 공선성 정리

### 실제 스피치

“예측 모델에서 먼저 본 것은 공선성입니다.  
원 변수와 계절조정 변수, 여러 시차·추세 파생변수가 함께 들어 있으면 회귀계수 해석이 흔들릴 수 있기 때문입니다.  
대표적으로 자전거 수요 변화량과 계절조정 수요 변화량의 절대 상관계수는 0.9349였고, 이 정도면 사실상 같은 정보를 중복해서 들고 있다고 볼 수 있습니다.  
그래서 중복 타깃 계열 변수, 장기 시차 변수, 추세 파생변수, 코드형 변수 등을 제거해 최종 입력 구조를 단순화했습니다.  
막대차트도 같은 결론을 보여주는데, 소수 변수쌍이 유난히 높게 튀어 있다는 것은 일부 파생변수가 사실상 같은 신호를 반복 전달하고 있다는 뜻입니다.”

### 유의점

- 핵심 숫자는 `0.9349` 하나만 강조한다.
- 제거 표는 모두 읽지 말고 제거 이유만 설명한다.
- 차트 해석은 `중복 신호가 뚜렷했다`는 결론으로 닫는다.

## 페이지 16-20. 누수 점검과 sample_weight 설계

### 실제 스피치

“다음은 높은 성능이 누수 때문이 아닌지 확인하는 단계입니다.  
직전 시점 변화량, 24시간 평균·표준편차, 168시간 평균·표준편차는 모두 과거값에서만 만들어졌는지 검증했고, mismatch는 0이었습니다.  
또 학습, 검증, 테스트는 2023, 2024, 2025로 시간순 분리되어 있습니다.  
그 다음으로는 월별 패턴이 너무 비슷하게 반복되는 문제를 봤습니다.  
예를 들어 대여 건수 기준 2023년 5월과 6월은 상관이 0.9947로 매우 유사했고, 이런 구간은 학습 가중치를 조금 낮춰 반복 패턴이 학습을 과도하게 지배하지 않도록 조정했습니다.  
히트맵은 이런 월별 반복이 일부 변수에서 일회성이 아니라 구조적으로 이어진다는 점을 보여주고, 뒤의 라인차트는 실제 일중 패턴이 거의 겹친다는 점을 직관적으로 보여줍니다.”

### 유의점

- `누수 없음`, `분할 이상 없음`, `sample_weight 필요성` 이 세 줄기로 설명한다.
- 히트맵은 월별 반복 구조의 증거라는 점만 짚는다.
- 라인차트는 `두 달이 비슷하다`보다 `같은 형태가 여러 달 반복된다`는 해석으로 연결한다.
- 가중치 설계는 성능 튜닝보다 `학습 데이터 대표성 균형 조정`으로 설명한다.

## 페이지 21-23. 모델링과 성능 결과

### 실제 스피치

“모델은 Ridge(alpha=2.0)이고, 전처리는 median imputation과 standardization을 사용했습니다.  
평가 지표는 RMSE, MAE, R²입니다.  
결과는 학습 R² 0.726576, 검증 R² 0.721155, 테스트 R² 0.727374로 split 간 차이가 크지 않았습니다.  
즉 과적합이 심하지 않으면서도 설명 가능한 기준 모델로 볼 수 있습니다.  
분할 규모 차트에서도 특정 연도 구간이 지나치게 작거나 비정상적으로 비어 있지 않아, 성능 비교의 전제가 안정적이라는 점을 함께 확인할 수 있습니다.”

### 유의점

- 여기서는 수치 3개만 읽는다.
- `최고 성능`보다 `안정적 기준 모델`이라는 표현을 유지한다.
- 성능 막대차트는 `학습만 높다`가 아니라 `세 구간이 비슷하다`는 점을 읽는다.

## 페이지 24-28. 회귀계수, LightGBM 비교, 클러스터별 결과

### 실제 스피치

“회귀계수 기준으로 보면 가장 영향력이 큰 변수는 대여 건수이고, 그다음은 직전 시점 변화량, 24시간 평균 변화량, 168시간 평균 변화량입니다.  
즉 현재 시점 대여량과 최근 이력 정보가 핵심 설명 변수라는 뜻입니다.  
회귀계수 차트에서 막대 길이가 긴 변수일수록 영향력이 크고, 양수와 음수 방향은 수요 변화량을 늘리는지 줄이는지를 해석하는 기준이 됩니다.  
한편 LightGBM은 원래 매우 높은 점수를 보였지만, 타깃 이력 파생변수를 제거하자 성능이 크게 하락했습니다.  
그래서 LightGBM의 높은 점수는 강한 과거 이력 피처와 반복되는 시계열 패턴의 영향이 컸다고 해석했습니다.  
또 군집별 Ridge 결과를 보면 군집마다 예측 난이도 차이가 있었고, 특히 cluster 1은 상대적으로 성능이 낮아 군집 특화 피처 보강이 필요한 구간으로 볼 수 있습니다.  
군집별 차트는 어느 군집이 더 설명 가능한 패턴을 갖는지, 반대로 어떤 군집이 외생 변수나 세분화 피처를 더 요구하는지를 보여주는 해석 근거입니다.”

### 유의점

- 계수는 상위 3~4개만 읽는다.
- LightGBM 비교는 `왜 높았는가` 해석으로 연결한다.
- 군집별 결과는 난이도 차이만 짚는다.
- 계수 차트에서는 변수명을 읽기보다 `현재 상태 + 최근 이력` 축이 중요하다고 묶어 말한다.
- 군집별 R²와 RMSE는 숫자보다 `군집별 난이도 격차 존재`라는 메시지로 정리한다.

## 페이지 29. 결론

### 실제 스피치

“정리하면, 군집화 단계에서는 대여소의 공간 역할을 설명할 수 있는 구조를 확보했고, 예측 단계에서는 그 구조를 시간대 수요 변화량 예측으로 연결하는 설명 가능한 Ridge 기준 모델을 만들었습니다.  
마지막 성능을 보면 학습 RMSE 0.726203, MAE 0.481299, R² 0.726576, 검증 R² 0.721155, 테스트 R² 0.727374 수준입니다.  
즉 이 모델은 높은 점수만을 노린 모델이라기보다, 누수 없이 안정적이고 해석 가능한 기준 모델로 의미가 있습니다.”

### 유의점

- 마지막은 `군집화와 예측이 하나의 흐름`이라는 점으로 닫는다.
- 결론에서는 성능표를 한 번 더 짧게 읽고 끝낸다.
