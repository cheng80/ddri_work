# 강남구 따릉이 통합 분석 보고서 v1 스피치 노트

작성일: 2026-03-19  
대상 문서: `01_ridge_clustering_integrated_report_v1.md`  
용도: 발표용 문서와 분리된 스피치 가이드

## 사용 원칙

- 발표 슬라이드형 문서에는 `결론`, `핵심 수치`, `그림 해석 포인트`만 남긴다.
- 배경 설명, 연결 문장, 강조 포인트는 이 스피치 문서에서 소화한다.
- 변수명은 필요할 때만 짧게 읽고, 기본적으로는 한글 의미 중심으로 설명한다.

## 도입

### 실제 스피치

“이번 보고서는 군집화와 예측을 각각 따로 설명하던 자료를 하나의 흐름으로 정리한 통합본입니다.  
먼저 대여소가 어떤 공간 역할을 갖는지 군집화로 해석하고, 그 다음 그 구조를 바탕으로 시간대별 수요 변화량을 예측하는 Ridge 분석으로 연결했습니다.  
핵심은 높은 점수 자체보다, 설명 가능한 구조를 가진 기준 모델을 확보하는 데 있습니다.”

### 유의점

- 첫 도입에서는 `군집화 -> 예측 연결`만 잡고 들어간다.
- `DDRI` 같은 내부 약어는 말하지 않는다.

## 1. 군집화에서 예측으로 이어지는 분석 흐름

### 실제 스피치

“먼저 왜 군집화가 먼저였는지 설명드리겠습니다.  
대여량이 많고 적음만으로는 각 대여소가 업무 중심지인지, 주거지인지, 상권 혼합지인지 설명하기 어렵습니다.  
그래서 반납 시간대 비율, 순유입, 교통 접근성을 이용해 대여소의 공간 역할을 먼저 분류했습니다.  
이렇게 역할 구조를 정리해두면, 예측 단계에서도 어떤 피처가 실제 수요 변동을 설명하는지 더 해석 가능하게 볼 수 있습니다.”

### 유의점

- 군집화는 예측 이전의 `구조 파악 단계`라는 점을 먼저 강조한다.
- 군집화 자체가 최종 목적이 아니라는 점을 유지한다.

## 2. 군집화 분석에서 사용한 기준

### 실제 스피치

“군집화 타겟은 station-level입니다.  
즉 하루 수요를 바로 맞추는 게 아니라, 각 대여소가 어떤 공간 역할을 갖는지 먼저 분류하는 작업입니다.  
메인 피처는 오전 반납 비율, 점심 반납 비율, 저녁 반납 비율, 아침 순유입, 저녁 순유입, 최근접 지하철 거리, 300미터 내 버스정류장 수의 7개입니다.  
이 피처 조합은 도착 시간대 성격과 이동 방향, 교통 접근성을 함께 보게 해 줍니다.”

### 유의점

- 변수명보다 의미를 읽는다.
- 7개를 다 나열하더라도 짧게 끊어서 읽는다.

## 3. 예측 분석 문제 정의

### 실제 스피치

“군집화는 역할을 설명하지만, 특정 시점의 수요 변화를 직접 예측하지는 못합니다.  
운영 관점에서는 결국 어느 시간대에 어느 대여소의 자전거가 부족해질지 알아야 하기 때문에, 다음 단계로 예측 분석이 필요합니다.  
이번 예측 타깃은 자전거 수요 변화량이고, 단위는 station-hour입니다.”

### 유의점

- 운영 활용 관점으로 자연스럽게 넘어간다.
- `station-hour`를 한 번 짚고 지나간다.

## 4. Ridge end-to-end 분석 목적과 데이터 구성

### 실제 스피치

“Ridge 분석의 목표는 자전거 수요 변화량을 설명 가능한 방식으로 예측하는 것입니다.  
흐름은 데이터 정제, 공선성 정리, 누수 점검, 시계열 패턴 진단, sample weight 적용, Ridge 학습, 성능 검증 순서입니다.  
사용 데이터는 2023~2024 학습, 2025 테스트 기준이며, 최종 입력 변수는 16개입니다.”

### 유의점

- 이 페이지에서는 전체 파이프라인 순서만 잡는다.
- 변수 표는 모두 읽지 말고 `기상 + 시간 + 과거 이력 + 군집` 정도로 묶어서 설명한다.

## 5. 데이터 정제와 공선성 정리

### 실제 스피치

“예측 모델에서 첫 번째로 본 건 공선성입니다.  
원 변수와 계절조정 변수, 여러 시차·추세 파생변수가 함께 들어 있으면 회귀계수 해석이 흔들릴 수 있기 때문입니다.  
대표적으로 자전거 수요 변화량과 계절조정 수요 변화량의 절대 상관계수는 0.9349였고, 이 정도면 사실상 같은 정보를 중복해서 들고 있다고 봤습니다.  
그래서 중복 타깃 계열 변수, 장기 시차 변수, 추세 파생변수, 역할이 겹치는 코드형 변수를 제거해 최종 입력 구조를 단순화했습니다.”

### 유의점

- 핵심 수치는 `0.9349` 하나만 강조한다.
- 제거 컬럼 표는 모두 읽지 않고 `왜 제거했는가` 중심으로 간다.

## 6. 누수 점검과 sample_weight 설계

### 실제 스피치

“다음으로는 높은 성능이 누수 때문이 아닌지 먼저 점검했습니다.  
특히 직전 시점 수요 변화량, 24시간 평균·표준편차, 168시간 평균·표준편차는 모두 과거값에서만 만들어졌는지 검증했고, 결과적으로 mismatch는 0이었습니다.  
또 학습, 검증, 테스트는 2023, 2024, 2025로 시간순 분리되어 겹치지 않았습니다.  
그 다음으로는 월별 유사패턴이 너무 반복되면 특정 구간이 과도하게 학습되는 문제를 보정하기 위해 sample weight를 적용했습니다.  
예를 들어 대여 건수 기준 2023년 5월과 6월은 상관이 0.9947로 매우 유사해서, 이런 구간은 학습 가중치를 조금 낮추는 방식으로 조정했습니다.”

### 유의점

- 누수 없음, split 이상 없음, sample weight 필요성 이 세 줄기로 설명한다.
- `corr=0.9947`은 월 유사패턴 설명 때만 짧게 쓴다.

## 7. 모델링과 성능 결과

### 실제 스피치

“모델은 Ridge(alpha=2.0)이고, 전처리는 median imputation과 standardization을 사용했습니다.  
학습에는 sample weight를 적용했고, 평가 지표는 RMSE, MAE, R2입니다.  
결과는 train R2가 0.726576, valid R2가 0.721155, test R2가 0.727374로 split 간 차이가 크지 않았습니다.  
즉 과적합이 심하지 않으면서도 설명 가능한 기준 모델로 볼 수 있습니다.”

### 유의점

- 이 페이지에서는 수치 3개만 읽고 지나간다.
- `최고 성능`보다 `안정적 기준 모델`이라는 표현을 유지한다.

## 8. 회귀계수, LightGBM 비교, 클러스터별 결과

### 실제 스피치

“회귀계수 기준으로 보면 가장 영향력이 큰 변수는 대여 건수이고, 그다음은 직전 시점 수요 변화량, 24시간 평균 변화량, 168시간 평균 변화량입니다.  
즉 현재 시점 대여량과 최근 이력 정보가 여전히 핵심 설명 변수라는 뜻입니다.  
한편 LightGBM은 원래 매우 높은 점수를 보였지만, 타깃 이력 파생변수를 제거하자 R2가 크게 하락했습니다.  
그래서 LightGBM의 높은 성능은 순수한 일반화 능력뿐 아니라 강한 과거 이력 피처와 반복되는 시계열 패턴의 영향이 컸다고 해석했습니다.  
클러스터별 Ridge 결과를 보면 군집마다 예측 난이도 차이가 있었고, 특히 cluster 1은 상대적으로 성능이 낮아 군집 특화 피처 보강이 필요한 구간으로 볼 수 있습니다.”

### 유의점

- 계수 표는 상위 3~4개만 읽는다.
- LightGBM 비교는 `왜 높은 점수가 나왔는가` 해석으로 연결한다.

## 9. 결론과 다음 단계

### 실제 스피치

“정리하면, 군집화 단계에서는 대여소의 공간 역할을 설명할 수 있는 구조를 확보했고, 예측 단계에서는 그 구조를 시간대 수요 변화량 예측으로 연결하는 설명 가능한 Ridge 기준 모델을 만들었습니다.  
현재 결과만으로도 군집화와 예측을 하나의 스토리로 연결하는 1차 통합본은 가능하다고 판단합니다.  
다음 단계에서는 공선성 파트를 v3 기준으로 다시 맞추고, 실험 설계와 한계, 평가 지표 설명을 더 보강해 최종 발표본으로 다듬는 방향이 적절합니다.”

### 유의점

- 마지막은 `군집화와 예측이 하나의 흐름`이라는 점으로 닫는다.
- 2차 보강 항목은 짧게만 언급한다.
