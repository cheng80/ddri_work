# HMW3 대여소 머신러닝 전체 분석 보고서

## 1. 보고서 개요

이 문서는 `hmw3` 폴더 안의 `Data`, `Note`, `scripts`를 근거로 상위 20개 대여소의 시간대별 대여량과 반납량 예측 결과를 전체 흐름 기준으로 정리한 보고서다.

- 분석 대상 대여소 수: `20개`
- 최고 성능 대여소: `2348 포스코사거리(기업은행)`
- 최고 통합 테스트 R²: `0.614`
- 전체 평균 통합 테스트 R²: `0.364`
- 전체 중앙값 통합 테스트 R²: `0.333`
- 평균 대여량 테스트 R²: `0.350`
- 평균 반납량 테스트 R²: `0.378`

- station별 행 수는 대부분 `26,304`행으로 동일하다.
- 음수 대여량/반납량은 집계되지 않았다.
- 시간 중복도 확인되지 않았다.

## 4. 전처리와 데이터 분할

노트북과 산출물 기준으로 전처리 규칙은 다음과 같다.

1. 시간 변수에서 `year`, `month`, `day`, `weekday`, `hour`를 분리한다.
2. 공휴일 기준표를 이용해 휴일 여부를 표시한다.
3. 주말과 공휴일을 묶어 `offday`로 처리하고, 나머지는 `weekday`로 둔다.
4. 연도 기준으로 `2023=train`, `2024=valid`, `2025=test` 구간을 고정한다.

이 구조는 시간 누수 없이 과거-현재-미래 순서를 유지하면서 alpha를 검증할 수 있다는 장점이 있다.

## 5. 패턴 기반 feature 설계

이번 프로젝트의 핵심은 원천 count를 바로 넣는 것이 아니라, 시간 패턴을 먼저 근사한 뒤 Ridge로 보정하는 구조라는 점이다.

주요 feature는 다음과 같다.

- `base_value`: day type별 기본 시간 패턴의 기준값
- `month_weight`: 월별 계절성 가중치
- `year_weight`: 연도별 보정치
- `hour_weight`: 시간대별 추가 보정치
- `pattern_prior`: 기본 패턴과 가중치를 합친 1차 사전값
- `corrected_pattern_prior`: Ridge 입력으로 쓰이는 보정된 핵심 패턴값

주요 특징은 다음과 같다.

- 즉, 대여소마다 과적합 방지가 필요한 정도가 다르고, 하나의 공통 alpha로 묶기 어렵다.

그 다음은 `pattern_prior`, `hour_weight`, `month_weight`, `base_value` 순으로 이어진다.

핵심 해석은 다음과 같다.

- 현재 모델은 장기 추세보다 "기존 패턴의 정교한 보정"에 더 강하다.
- `year_weight` 비중이 거의 0에 가까워, 연도별 구조 변화는 잘 포착하지 못한다.
- 시간대와 계절성이 성능을 좌우하는 중심 요인이다.

## 11. 종합 해석

이번 프로젝트는 "패턴형 수요를 가진 station"에 대해서는 꽤 설득력 있는 결과를 보여준다.  
특히 상위권 대여소는 통합 테스트 R²가 `0.40~0.61` 수준으로, 운영 참고용 수요 예측으로는 충분히 의미가 있다.

반면 한계도 분명하다.

1. 급격한 이벤트성 수요는 과소추정되는 경향이 있다.
2. 출퇴근 피크 시간대 오차가 반복된다.
3. 연도별 구조 변화는 `year_weight`만으로 거의 설명되지 않는다.
4. 하위권 대여소는 패턴 불안정성이 높아 일반화 성능이 낮다.

## 12. 개선 제안

향후 성능을 높이려면 다음 방향이 유효하다.

1. 날씨, 강수, 기온, 미세먼지 같은 외생 변수를 추가한다.
2. 행사일, 학사 일정, 주변 교통 이벤트 등 특수일 feature를 만든다.
3. 출퇴근 피크 시간대 전용 보정 모델을 별도로 둔다.
4. station 군집별로 alpha와 feature 구성을 다르게 가져간다.
5. 장기 추세를 반영할 수 있는 연속 시계열 모델이나 boosting 계열 모델과 비교한다.
