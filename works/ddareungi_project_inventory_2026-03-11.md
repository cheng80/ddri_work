## 4. 문서와 실제 데이터의 차이

기획 문서와 군집화 문서의 방향은 맞지만, 예시 코드의 입력 스키마는 실제 데이터와 다름.

### 군집화 문서 예시 입력
- `station_id`
- `rent_datetime`

### 실제 따릉이 이용 데이터 컬럼
- `대여일시`
- `대여 대여소번호`
- `대여거치대`
- `반납일시`
- `반납대여소번호`
- `반납대여소명`
- `반납거치대`
- `이용시간(분)`
- `이용거리(M)`
- `생년`
- `성별`
- `이용자종류`

따라서 군집화용 노트북/코드는 실제 한글 컬럼 기준으로 다시 작성해야 한다.

## 6. 군집화 1차 시각화에 바로 쓸 수 있는 최소 조합

### 필수
- 따릉이 이용 이력
- 강남구 대여소 위치 정보

이 두 개만으로 아래 작업이 가능하다.
- 대여소별 평균 대여량 계산
- 평일/주말 평균 대여량 계산
- 출퇴근/야간 비율 계산
- K-Means 군집화
- PCA 2D 시각화
- 지도 위 군집 색상 시각화

### 해석 강화용
- 공원 정보
- 지하철역 정보
- 버스정류소 위치 정보

이 데이터는 군집 생성보다 군집 설명에 우선 사용한다.

## 7. 지금 시점의 가장 현실적인 1차 목표

`군집화 시각화 자료`의 1차 버전은 아래 범위로 잡는 것이 적절함.

1. 2023~2025 강남구 이용 이력 통합
2. 대여소별 패턴 Feature 생성
3. K=3~4 후보 비교
4. Elbow / Silhouette 시각화
5. PCA 산점도 시각화
6. 강남구 지도 위 군집 결과 표시
7. 군집별 평균 Feature 표 작성

환경 데이터는 1차 결과 해석용으로 후속 결합

## 8. 즉시 주의할 점

- 컬럼명이 대부분 한글이라 전처리 단계에서 영문 alias를 따로 만드는 것이 좋음
- 지하철/버스 데이터는 인코딩 이슈가 있어 `cp949` 처리 필요
- 버스 데이터는 좌표 컬럼명이 통일되어 있지 않음
- 생활인구 데이터는 행정동 단위이므로 대여소 좌표와 바로 결합되지 않음
- 학습/테스트 분리를 명확히 고정해야 함
  - 학습: 2023~2024
  - 테스트: 2025
- 향후 파일명, 노트북명, 산출물명은 `ddri_*` 규칙으로 통일

## 10. 결론

현재 데이터 상태는 “플랜만 있는 단계”가 아니라, 이미 군집화 1차 결과를 만들 수 있는 수준이다.

가장 먼저 만들 산출물은 다음이 적절하다.

- 강남구 대여소별 이용 패턴 Feature 테이블
- K-Means 군집화 결과
- PCA 시각화
- 지도 시각화

즉, 다음 실작업은 전체 플랜 작성보다 먼저 `군집화 1차 노트북`을 만드는 것이 가장 효율적이다.
