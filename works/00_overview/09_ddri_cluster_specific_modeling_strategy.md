# Ddri Cluster-Specific Modeling Strategy

작성일: 2026-03-14  
목적: 군집화 이후 예측 파트에서 대표 대여소 단계와 전체 스테이션 baseline 단계의 의미를 구분하고, 이후 군집별 최적 피처/모델 탐색으로 어떻게 넘어갈지 정리한다.

## 1. 왜 이 문서가 필요한가

현재 프로젝트는 이미 군집화를 완료했고, 예측 파트에서도 여러 데이터셋과 baseline 실험을 진행했다.

하지만 질문이 하나 생긴다.

- 군집을 나눈 이유가 결국 군집별로 다른 피처 조합과 다른 모델을 찾기 위한 것이라면,
- 대표 대여소 15개 실험은 어디에 해당하는가

이 문서는 그 위치를 분명히 하기 위해 만든다.

## 2. 현재 해석의 핵심

군집화의 최종 목적은 단순 분류 자체가 아니다.

핵심 목적은 아래와 같다.

- 이용 패턴이 비슷한 스테이션을 같은 그룹으로 묶고
- 군집별 수요 구조 차이를 해석하고
- 이후 군집별로 더 적합한 피처/모델/운영 규칙을 검토할 수 있게 하는 것

즉, 군집은 `최종 예측 전략을 더 세분화하기 위한 구조`다.

## 3. 대표 대여소 15개 단계의 의미

대표 대여소 15개 `station-hour` 실험은 군집별 최적화의 최종 단계가 아니다.

이 단계의 역할은 아래와 같다.

### 3.1 설명용 역할

- 각 군집이 실제로 어떤 시간대 패턴을 보이는지 사람이 이해하기 쉽게 보여준다.
- 발표나 보고서에서 군집별 사례를 설명할 수 있게 한다.

### 3.2 탐색용 역할

- 전체 161개 스테이션에 앞서, 시간 단위 예측 구조가 대체로 작동하는지 빠르게 확인한다.
- 추천 모델 문서에서 제안된 모델 후보를 작은 실험셋에서 먼저 점검한다.

### 3.3 실험 설계용 역할

- 이후 전체 스테이션 실험이나 군집별 본실험에서 어떤 피처와 어떤 validation 전략을 쓸지 감을 잡는 데 사용한다.

즉, 대표 대여소 단계는 `군집별 본격 최적화 전의 설명/탐색/실험 설계 단계`다.

## 4. 전체 161개 baseline 단계의 의미

전체 공통 스테이션 161개 `station-hour` 실험은 대표 대여소 단계와 다르게, 실제 서비스 확장성을 확인하는 본 baseline 단계다.

이 단계의 목적은 아래와 같다.

- 전체 서비스 범위에서 공통적으로 작동하는 baseline을 확보
- 운영자/일반 사용자 기능으로 연결 가능한 공통 입력 구조 확인
- 군집별 분리 없이도 어느 정도 성능이 나오는지 기준선 확보

즉, 이 단계는 `서비스용 공통 baseline`이다.

## 5. 아직 하지 않은 진짜 다음 단계

군집을 나눈 이유를 예측 모델링에 끝까지 반영하려면, 다음 단계가 추가로 필요하다.

### 5.1 군집별 subset 분리

- 각 `cluster`별로 전체 스테이션을 나눈다.
- 예: 업무/상업 혼합형, 아침 도착 업무 집중형, 주거 도착형 등

### 5.2 군집별 피처 전략 비교

군집마다 중요도가 다를 수 있는 피처를 비교한다.

예:

- 업무형:
  - `hour`, `weekday`, 출퇴근 패턴 lag
- 주거형:
  - `weekend`, `holiday`, 야간/주말 패턴
- 생활권 혼합형:
  - `weather`, `lag_24h`, `lag_168h`, 생활권 반복성

### 5.3 군집별 모델 전략 비교

군집마다 다른 모델이 더 맞을 수 있는지 본다.

예:

- 어떤 군집은 `LightGBM`이 가장 안정적일 수 있음
- 어떤 군집은 더 단순한 선형 구조도 충분할 수 있음
- 어떤 군집은 `Poisson` 계열 objective가 상대적으로 유리할 수 있음

### 5.4 군집별 운영 규칙 연결

예측모델이 같더라도, 운영 해석은 군집별로 다를 수 있다.

예:

- 업무형은 오전 부족 리스크 임계값을 더 민감하게
- 주거형은 야간/주말 회수 기준을 다르게
- 생활권 혼합형은 시간대별 재배치 우선순위 규칙을 완화 또는 강화

## 6. 현재까지의 단계 위치

현재 프로젝트의 예측 파트는 아래 순서까지 진행된 상태다.

1. 군집화 완료
2. 대표 대여소 15개 선정
3. 대표 대여소 `station-hour` 탐색 실험 완료
4. 전체 161개 `station-hour` 공통 baseline 완료
5. 전체 161개 `station-hour` objective 비교 완료

아직 아래 단계는 미진행이다.

6. 군집별 subset 기반 모델 분리 실험
7. 군집별 최적 피처 조합 비교
8. 군집별 최적 모델 비교
9. 군집별 운영 규칙 차등화

즉, 현재는 `군집별 모델링으로 넘어가기 직전 단계`까지 와 있다.

## 7. 권장 다음 순서

다음 작업 순서는 아래가 적절하다.

참고:

- `상위 오류 스테이션`은 스테이션별 `MAE` 또는 `RMSE`가 큰 순서대로 봤을 때 상위에 오는, 즉 모델이 상대적으로 잘 못 맞추는 스테이션을 뜻한다.

1. 전체 161개 `station-hour` 상위 오류 스테이션 분석
2. 오류가 큰 스테이션이 특정 군집에 몰리는지 확인
3. 군집별 subset 데이터셋 설계
4. 군집별로 피처 조합 2~3개 비교
5. 군집별로 모델 1~2개 비교
6. 군집별 운영 해석 규칙 초안 작성

## 8. 현재 결론

현재까지의 해석은 아래처럼 고정한다.

- 대표 대여소 15개 단계는 군집별 본격 최적화 전에 수행한 설명/탐색 단계다.
- 전체 161개 `station-hour` baseline은 서비스용 공통 기준선을 확보하는 단계다.
- 군집을 나눈 진짜 목적을 예측 파트까지 완성하려면, 다음에는 `군집별 subset 기반 모델링`으로 넘어가야 한다.
