# Ddri Notebook And Evidence Chart Policy

작성일: 2026-03-12  
목적: 이 프로젝트에서 노트북 문서화 수준과 근거 차트 생성 기준을 고정하여, 이후 산출물이 같은 품질과 같은 설명 방식을 유지하도록 한다.

## 1. 기본 원칙

- 노트북은 단순 실행 파일이 아니라 `읽는 사람이 따라갈 수 있는 실험 문서`로 작성한다.
- 스크립트는 재생성 기준이고, 노트북은 설명 기준이다.
- 앱 구현 전까지의 모든 분석과 ML 작업은 원칙적으로 노트북 기반으로 진행한다.
- 전처리, feature 생성, 모델 학습, 평가, 비교 실험, 오류 사례 분석은 모두 노트북에서 추적 가능해야 한다.
- 보고서/발표용 차트는 `보기 좋은 차트`보다 `왜 이 기준을 선택했는지 설명하는 근거 차트`를 우선한다.
- 없는 자료는 임의 생성하지 않으며, 근거가 약한 해석은 보수적으로 표현한다.

## 2. 노트북 작성 정책

### 2.1 모든 대표 노트북에 반드시 들어갈 항목

- 문서 목적
- 분석 범위
- 입력 데이터 경로 또는 입력 데이터 설명
- 전처리 기준
- 사용 변수 또는 생성 feature 설명
- 실험 조건 또는 모델 설정
- 평가 기준과 비교 대상
- 생성 산출물 목록
- 해석 시 주의점

### 2.2 마크다운 셀 작성 기준

- 각 주요 코드 블록 앞에는 `왜 이 작업을 하는지`를 설명하는 마크다운 셀을 둔다.
- 결과 표나 차트 뒤에는 `이 결과를 어떻게 읽어야 하는지`를 짧게 정리한다.
- 섹션 제목만 두고 설명이 비어 있는 형태는 지양한다.
- 분석 단계가 바뀌면 새 마크다운 셀로 흐름을 끊어 준다.

### 2.3 코드 셀 주석 기준

- 입력 데이터 로드 셀에는 사용하는 파일과 컬럼 목적을 짧게 적는다.
- 전처리 셀에는 어떤 값이 제거 또는 유지되는지 주석을 넣는다.
- feature 생성 셀에는 각 파생 변수의 의미를 주석으로 적는다.
- 모델/군집화 셀에는 선택한 기법과 핵심 파라미터 이유를 주석으로 남긴다.
- 저장 셀에는 어떤 파일이 어떤 목적의 정본인지 주석으로 남긴다.

## 3. 근거 차트 정책

### 3.1 차트 선정 원칙

- 차트는 `결과 소개`보다 `선정 근거 설명`에 우선 배치한다.
- 하나의 주장에는 가능하면 하나 이상의 정량 차트를 붙인다.
- 발표용 차트는 `한글(영문)` 표기를 기본으로 한다.
- 차트 제목, 축, 범례, 피처명, 범주명은 가능하면 `한글(영문)` 형식으로 통일한다.
- 관련 README, 리포트 로그, 문서 본문에서도 용어를 `한글(영문)` 형식으로 맞춘다.
- `koreanize_matplotlib` 기반으로 한글을 우선 처리하고, seaborn/theme 설정이 이를 덮어쓰지 않도록 순서를 점검한다.

### 3.2 우선 생성 대상 차트

#### 전처리

- 전처리 전후 행 수 비교
- 제거 사유별 건수 비교
- 중복 로그 비율
- IQR 기준 극단치 후보 수

#### 군집화

- Elbow plot
- 군집 수별 Silhouette 비교
- PCA 산점도
- 군집별 주요 특성 평균 비교
- 입력 특성 상관관계 히트맵
- 군집 프로파일 히트맵
- 군집별 대여소 수
- 월별 대여 건수 추이
- 요일-시간대 히트맵

#### 환경 해석

- 군집별 공원/지하철/버스 접근성 비교
- 대표 대여소 표
- 지도 시각화

#### 예측 데이터셋

- self-return / return / net_flow 요약 차트
- same_station_return_ratio 분포
- target-날씨-운영지표 상관관계 히트맵
- 휴일/비휴일, 주중/주말 비교 차트
- 월별 대여량 추이

### 3.3 차트 사용 판단 기준

- `컨퓨전 매트릭스`는 분류 문제일 때만 사용한다.
- 현재 프로젝트의 군집화와 수요예측 회귀 단계에서는, 컨퓨전 매트릭스보다 상관관계 히트맵, 분포 비교, 시계열 추이 차트가 우선이다.
- 설명력이 낮은 차트는 많이 만들기보다, 발표 문장과 직접 연결되는 차트를 남긴다.

## 4. 문서 반영 정책

- 새 차트가 만들어지면 관련 노트북의 마크다운 설명에도 반영한다.
- 발표 문서와 스피치 노트는 실제 사용한 차트 기준으로 갱신한다.
- `02_ddri_project_report_log.md`에는 차트 생성 이유와 해석 메시지를 함께 남긴다.
- 대표 노트북과 대표 스크립트는 각 파트 `README.md`에서 바로 열 수 있게 유지한다.

현재 군집화 대표 노트북:

- `works/01_clustering/08_integrated/pipeline/12_ddri_integrated_clustering_report_builder.ipynb`

현재 군집화 최종 산출물 경로:

- `works/01_clustering/08_integrated/final/`

## 5. 앞으로의 적용 방식

- 군집화, 전처리, 예측 데이터셋 노트북은 이 정책에 맞춰 지속 보강한다.
- 이후 ML 베이스라인, 고도화 모델, 시간대 예측 실험도 모두 노트북 중심으로 남긴다.
- 새 분석 파트가 생기면 먼저 대표 노트북과 재생성 스크립트를 짝으로 만든다.
- 이후 생성되는 차트는 목적별 폴더에 저장하고, 노트북/발표 문서/레포트 로그에 연결한다.
