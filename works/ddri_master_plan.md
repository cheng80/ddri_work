## 2. Phase 1. 데이터 자산 확정

목표: 실제 사용 가능한 데이터와 사용 불가능한 데이터를 구분하고, 이후 단계의 입력 기준을 고정한다.

### 체크리스트

- [x] 원천 데이터 폴더 구조 확인
- [x] 핵심 데이터 스키마 샘플 확인
- [x] 학습/테스트 연도 분리 기준 확정
- [x] `works/` 아래 실행용 폴더 구조 생성
- [ ] 데이터 인벤토리 문서명 `ddri_*` 규칙으로 정리 여부 결정
- [x] 군집화 1차 작업에 사용할 입력 파일 목록 정리
- [x] 연도별 대여소 기준 정보 사용 전략 확정
  - 2023~2025 공통 169개 대여소를 1차 군집화 기준으로 사용

### 완료 기준

- 실제 사용할 입력 파일 목록이 문서로 고정되어 있음
- 학습/테스트 기준이 문서로 고정되어 있음
- 폴더 구조가 생성되어 있음

### 막히면 요청할 자료

- 연도별 대여소 변경 이력 기준
- 2023~2025 중 기준 대여소 마스터를 어느 연도로 삼을지에 대한 팀 결정

---

## 3. Phase 2. 군집화 1차 산출물

목표: 강남구 대여소 이용 패턴 기반 군집화 결과를 1차 시각화까지 완성한다.

### 체크리스트

- [x] `works/clustering/` 폴더 생성
- [x] `works/clustering/data/` 폴더 생성
- [x] `works/clustering/images/` 폴더 생성
- [x] `works/clustering/ddri_station_clustering_baseline.ipynb` 생성
- [x] 2023~2024 학습용 따릉이 이용 이력 로드 코드 작성
- [x] 2025 테스트용 이용 이력 분리 로드 코드 작성
- [x] 실제 한글 컬럼 기준 전처리 코드 작성
- [x] 대여소별 군집화 feature 생성
  - avg_rental
  - weekday_avg
  - weekend_avg
  - peak_ratio
  - night_ratio
  - weekday_weekend_gap
  - rental_std
- [x] feature 저장 CSV 생성
- [x] StandardScaler 적용
- [x] KMeans 후보 K 탐색
- [x] Elbow plot 생성
- [x] Silhouette 비교 표 또는 그래프 생성
- [x] 최종 K 선택 근거 기록
- [x] PCA 2D 시각화 생성
- [x] 군집별 feature 평균표 생성
- [x] 군집 라벨 해석 초안 작성

### 완료 기준

- 군집 feature 테이블이 CSV로 저장됨
- 최소 3종 시각화가 저장됨
  - Elbow
  - PCA
  - 군집별 비교 차트
- 어떤 K를 선택했는지 근거가 문서/노트북에 남아 있음
- 발표용 지도 시각화 HTML이 생성됨

### 막히면 요청할 자료

- 군집 결과를 지도에 표시할 때 사용할 배경 지도/행정경계 기준
- 군집 라벨 명칭에 대한 팀 내부 합의

## 4. Phase 3. 예측용 데이터셋 설계

목표: 군집 결과를 포함한 예측용 학습 테이블 구조를 정의한다.

### 체크리스트

- [x] 예측 target 정의 확정
  - 1차 target은 `Station별 하루 대여량 (rental_count)`으로 고정
- [x] 예측 기준 테이블 grain 확정
  - 1차 grain은 `station-day`
- [x] 내부 데이터와 외부 데이터 결합 키 설계
- [x] 날씨 결합 기준 설계
- [x] 공휴일 결합 기준 설계
- [x] 환경 feature 결합 전략 설계
- [x] 군집 label을 예측 feature로 포함할지 결정
- [x] 학습용 최종 스키마 문서화

### 완료 기준

- 예측 단위와 타깃이 문서로 고정됨
- 결합 규칙이 문서로 정리됨
- 최종 학습 테이블 스키마 초안이 존재함

### 막히면 요청할 자료

- 팀이 최종 예측 대상으로 삼는 정확한 target 정의
- 웹에서 보여줄 예측값의 단위 정의

---

## 5. Phase 4. ML 베이스라인 구축

목표: 기본 성능을 확인할 수 있는 예측 모델 파이프라인을 만든다.

### 체크리스트

- [ ] 베이스라인 데이터셋 생성
- [ ] train/validation/test 분리 전략 확정
- [ ] baseline 모델 1개 이상 학습
  - Linear Regression 또는 RandomForest
- [ ] 트리 기반 모델 1개 이상 학습
  - XGBoost 또는 LightGBM
- [ ] 평가 지표 계산
- [ ] feature importance 또는 해석 자료 생성
- [ ] 2025 테스트 평가 결과 정리
- [ ] 오류 사례 분석

### 완료 기준

- 최소 2개 모델 결과 비교표 존재
- 2025 테스트셋 결과가 정리됨
- 베이스라인 대비 개선 여부가 보임

### 막히면 요청할 자료

- 최종 평가지표 우선순위
- 모델 결과를 웹에서 어떤 형태로 소비할지에 대한 요구사항
