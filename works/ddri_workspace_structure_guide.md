# Ddri 작업물 구조 가이드

작성일: 2026-03-12  
목적: 계획 문서 순서대로 작업물을 따라가기 쉽도록 `works` 폴더의 권장 구조와 저장 원칙을 정의한다.

## 1. 왜 구조를 바꾸는가

현재 작업물은 기능별로는 구분되어 있지만, 처음 보는 사람이 `어떤 문서를 먼저 열고`, `어떤 실험이 어떤 산출물을 만들었는지` 따라가기는 어렵다.

따라서 앞으로는 아래 원칙을 적용한다.

- 계획 문서 순서에 맞춰 상위 폴더를 번호로 구분
- 각 파트 안에서는 `baseline → 전처리 → 환경 해석 → 시각화 → 발표자료` 흐름으로 다시 분류
- 이미지도 한 폴더에 몰지 않고 목적별 파트 안에서 함께 관리
- 발표 직전에는 기존 경로를 무리하게 전부 이동하지 않고, 새 구조를 기준 경로로 삼아 점진 이관

## 2. 권장 상위 구조

### `works/00_overview`

- 전체 프로젝트를 이해하기 위한 문서
- 예:
  - 마스터 플랜
  - 프로젝트 레포트 로그
  - 데이터/산출물 인벤토리
  - target 정의 문서
  - 데이터셋 설계 문서

### `works/01_clustering`

- 군집화 파트를 처음부터 끝까지 따라가는 폴더

세부 구조:
- `01_baseline`
  - 군집화 노트북, 실행 스크립트
- `02_preprocessing`
  - 전처리 관련 문서, 로그, 요약표
- `03_environment`
  - 환경 feature 생성 스크립트와 요약 결과
- `04_maps`
  - Folium 지도, 정적 지도
- `05_presentation`
  - 군집화 발표용 MD, PDF, 스피치 노트
- `06_data`
  - 군집화 결과 CSV
- `07_images`
  - 군집화 차트 PNG

### `works/02_data_collection`

- 외부 데이터 수집 전용

세부 구조:
- `01_calendar`
  - 공휴일 API 수집 노트북, 스크립트, 결과
- `02_weather`
  - Open-Meteo 수집 노트북, 스크립트, 결과

### `works/03_prediction`

- 예측용 데이터셋과 모델링 준비 파트

세부 구조:
- `01_dataset_design`
  - 예측용 스키마, feature 설계 문서
- `02_data`
  - 학습/테스트 CSV, flow metric 요약
- `03_images`
  - 예측 관련 설명 차트
- `04_scripts`
  - 데이터셋 빌더, 차트 생성 스크립트

### `works/04_presentation`

- 최종 발표용 문서만 모아두는 폴더

세부 구조:
- `01_clustering`
  - 군집화 발표 자료
- `02_project`
  - 전체 프로젝트 발표 자료

## 3. 앞으로의 저장 원칙

### 문서

- 사람이 읽는 문서는 `목적별 폴더` 안에 저장
- 파일명 앞에는 필요하면 세부 순서를 붙임
  - 예: `01_problem_definition.md`
  - 예: `02_preprocessing_rules.md`

### 코드

- 노트북과 실행 스크립트는 같은 파트 안에서 함께 관리
- 재생성 기준이 되는 실행 스크립트는 파트 폴더의 앞쪽에 둔다

### 데이터

- CSV는 해당 파트 안의 `data` 또는 `06_data`에 저장
- 중간 산출물과 최종 산출물을 혼합하지 않도록 목적별 분리

### 이미지

- 차트는 차트를 만든 파트의 이미지 폴더에 저장
- 발표 전용 캡처본은 `presentation` 또는 `maps`와 분리 가능

## 4. 현재 권장 열람 순서

프로젝트 전체 흐름을 따라가려면 아래 순서가 가장 좋다.

1. `works/00_overview`
2. `works/01_clustering`
3. `works/02_data_collection`
4. `works/03_prediction`
5. `works/04_presentation`

## 5. 현재 적용 방식

발표 직전이라 기존 파일 경로를 한 번에 모두 이동하면 문서 링크, PDF, 발표 파일 경로가 깨질 수 있다.

따라서 현재는 다음 원칙으로 적용한다.

- 새 폴더 구조를 먼저 만든다.
- 앞으로 생성되는 산출물은 새 구조를 우선 사용한다.
- 기존 파일은 발표 이후 순차적으로 이관하고 링크를 다시 정리한다.

## 6. 발표 이후 권장 정리 작업

1. `works` 최상위 문서를 `00_overview`로 이동
2. `works/clustering` 파일을 `01_clustering` 하위 단계 폴더로 이동
3. `works/calendar`, `works/weather`를 `02_data_collection`로 통합
4. `works/prediction`을 `03_prediction` 구조로 재정리
5. `works/presentation`의 발표 파일을 `04_presentation`으로 이관
6. 문서 내부 링크를 새 경로 기준으로 일괄 수정
