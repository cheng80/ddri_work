# DDRI 군집별 2차 실험 판단 기준(Second-Round Experiment Criteria)

목적: `cluster00 ~ cluster04`의 1차 결과가 모였을 때, 어느 군집에 2차 실험을 추가할지 같은 기준으로 판단한다.

## 1. 2차 실험이 필요한 경우

아래 중 하나라도 해당하면 2차 실험 우선 후보로 본다.

- `LightGBM_RMSE`의 test RMSE가 다른 군집보다 눈에 띄게 높다
- `LinearRegression` 대비 개선 폭이 작다
- validation과 test 차이가 커서 일반화가 불안정하다
- 시간대 오차가 특정 구간에 몰린다
- 군집 특성상 날씨나 주말 패턴을 더 세밀하게 볼 필요가 있다

## 2. 2차 실험 우선순위

### 2.1 1순위

- test RMSE가 가장 높은 군집
- validation/test 괴리가 큰 군집

### 2.2 2순위

- `LightGBM_RMSE`가 분명히 우세하지만 오차가 여전히 큰 군집
- 피처 추가 효과를 볼 가능성이 큰 군집

### 2.3 3순위

- 이미 성능이 안정적인 군집
- 1차 결과만으로도 해석이 충분한 군집

## 3. 허용되는 2차 실험 종류

### 3.1 모델 추가

- `LightGBM_Poisson`
- `CatBoost_RMSE`
- `CatBoost_Poisson`

### 3.2 피처 추가

- 추가 lag
- 추가 rolling
- 상호작용용 범주 조합
- 날씨 파생 변수

### 3.3 피처 축소

- 효과가 약한 변수 제거
- 범주형 축소

## 4. 2차 실험 전 확인 사항

- 1차 공통 프로토콜 점수(`RMSE`, `MAE`, `R²`)가 모두 기록되었는가
- 우세 모델이 validation 기준으로 선택되었는가
- test를 보고 1차 모델을 고르지 않았는가
- 2차 실험 목적이 명확히 적혀 있는가

## 6. 현재 기본 원칙

- 1차 공통 비교가 끝나기 전에는 2차 실험으로 넘어가지 않는다
- 2차 실험은 모든 군집에 일괄 적용하지 않는다
- 가장 어려운 군집부터 우선적으로 확장한다
- 2차 실험 결과도 반드시 노트북과 취합표에 남긴다
