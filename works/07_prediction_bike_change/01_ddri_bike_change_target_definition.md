# DDRI Bike-Change 타깃 정의

## 1. 목적

기존 `rental_count(대여량)` 대신 `bike_change(시간별 자전거 변화량)`을 신규 예측 타깃으로 사용한다.

이번 타깃의 목적은 단순 수요량 예측이 아니라, 특정 시간에 대여소 자전거가 실제로 늘어나는지 줄어드는지를 더 직접적으로 반영하는 것이다.

## 2. 타깃 정의 원칙

- 타깃 이름: `bike_change`
- 단위: `station-hour(대여소-시간 단위)`
- 해석:
  - 양수: 직전 시간보다 자전거 수 증가
  - 음수: 직전 시간보다 자전거 수 감소
  - 0: 변화 없음

## 3. 현재 전제

이번 실험은 기존 `prediction_long` 실험 구조를 최대한 유지하면서, 타깃만 `bike_change`로 바꾸는 방식으로 시작한다.

즉 첫 라운드에서는:
- 데이터 범위: 대표 15개
- 학습/테스트 구분: 기존과 동일
- 피처 세트: 필터링된 `38개 top20_optional`을 기준으로 하되, rep15 입력에 없는 2개 피처를 제외한 `36개 사용 가능 피처`
- 모델: `LightGBM regression`

## 4. 타깃 관련 주의점

- `bike_change`는 음수가 가능하므로 `Poisson objective`는 1차 비교에서 제외한다
- 단순 RMSE뿐 아니라 방향성도 중요하므로 후속 단계에서 `부호 일치율`을 같이 볼 수 있다
- 기존 `lag`, `rolling` 피처는 우선 기존 방식과 최대한 맞추되, 필요하면 추후 `bike_change 기반 lag/rolling`으로 확장 비교한다

## 5. 현재 결정

- 1차 실험은 `기존 파이프라인 유지 + 신규 타깃 bike_change + rep15 사용 가능 36개 축소 피처`로 고정한다
- one-hot encoding은 이번 첫 라운드에서 제외하고, baseline 성능 확인 후 별도 비교한다
