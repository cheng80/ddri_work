# DDRI 시간·요일 원형 인코딩 추가 실험

작성일: 2026-03-17

## 1. 목적

`hour`, `weekday`는 숫자형으로 저장되어 있지만 실제로는 주기형 변수다.

예:

- `23시`와 `0시`는 실제 시간상 매우 가깝다.
- `일요일`과 `월요일`도 실제 순서상 붙어 있다.

하지만 숫자 자체만 쓰면 아래처럼 직선 거리로 해석될 수 있다.

- `23`과 `0`은 멀게 보임
- `6`과 `0`도 멀게 보임

따라서 `06_prediction_training`에서는 이를 보완하는 **원형 인코딩 비교 실험**을 추가한다.

## 2. 추가 피처

비교 대상 피처는 아래 4개다.

- `hour_sin`
- `hour_cos`
- `weekday_sin`
- `weekday_cos`

계산식:

- `hour_sin = sin(2π * hour / 24)`
- `hour_cos = cos(2π * hour / 24)`
- `weekday_sin = sin(2π * weekday / 7)`
- `weekday_cos = cos(2π * weekday / 7)`

## 3. 왜 sin, cos를 같이 쓰는가

`sin` 하나만 쓰면 서로 다른 시간대가 같은 값에 겹칠 수 있다.

그래서:

- `sin`은 원 위의 세로 축
- `cos`는 원 위의 가로 축

역할을 하며, 두 값을 같이 써야 주기 위치를 안정적으로 표현할 수 있다.

## 4. 현재 프로젝트에서의 위치

이 실험은 **정본 수정 실험이 아니라 학습 단계 피처 비교 실험**이다.

즉:

- `05_prediction_canonical`에는 넣지 않음
- `06_prediction_training`에서만 비교

원칙:

1. 정본 CSV는 그대로 유지
2. `modeling_data` 생성 또는 학습 전처리 단계에서만 4개 피처 추가
3. 무샘플링 기준선과 같은 학습 정책으로 비교

## 5. 비교 방식

비교는 아래 두 조건으로 나눈다.

### A. 기존 기준선

- `hour`
- `weekday`
- 나머지 기존 피처 그대로 사용

### B. 원형 인코딩 추가

- 기존 기준선 피처 전부 유지
- 추가로 아래 4개 포함
  - `hour_sin`
  - `hour_cos`
  - `weekday_sin`
  - `weekday_cos`

즉, 현재 계획은 `hour`, `weekday`를 제거하는 실험이 아니라,
**기존 피처에 원형 인코딩 4개를 추가하는 실험**이다.

## 6. 평가 기준

기존과 동일하게 아래 지표를 모두 저장한다.

- Validation `RMSE`
- Validation `MAE`
- Validation `R²`
- Test `RMSE`
- Test `MAE`
- Test `R²`

학습 정책도 동일하다.

- Train: `2023`
- Validation: `2024`
- Validation 기준 우세 모델 선택
- `2023+2024` 재학습
- Test: `2025`

## 7. 기대 효과와 주의점

기대 효과:

- 시간 경계(`23시 ↔ 0시`) 왜곡 완화
- 요일 경계(`일요일 ↔ 월요일`) 왜곡 완화
- 선형적 거리 해석 문제 보완

주의점:

- `LightGBM` 같은 트리 모델은 `hour`, `weekday` 정수값만으로도 어느 정도 처리할 수 있다.
- 따라서 개선 폭이 크지 않을 수 있다.
- 반드시 기존 기준선과 직접 비교해야 한다.
