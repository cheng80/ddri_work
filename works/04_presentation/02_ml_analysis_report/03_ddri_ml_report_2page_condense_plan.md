## 2. 페이지별 구성안

### 2.1 1장: 개요 + 문제 + 데이터 + 모델 + 핵심 결과

#### 1) 제목 + 목적 (2~3줄)
- **제목**: 강남구 따릉이 머신러닝 분석 보고서
- **목적**: 대여소별 시간대 수요 예측 실험 결과 정리, 군집화 → 대표 15개 탐색 → 161개 운영 기준선 선택

#### 2) 문제 정의 (불릿 3~4개)
- 예측 단위: **대여소·시간 단위** (특정 대여소, 특정 시각의 대여량)
- 활용: 잔여 자전거 예측, 재배치 우선순위 판단
- 평가 지표: 오차(RMSE), 평균오차(MAE), 설명력(R²)

#### 3) 데이터 범위 (표 1개, 4~5행)
| 구분 | 대표 15개 | 전체 161개 |
|------|-----------|------------|
| 역할 | 군집별 피처 탐색 | 운영 기준선 선택 |
| 학습 | 2023~2024 | 2023~2024 |
| 테스트 | 2025 | 2025 |

#### 4) 군집 + 모델 (불릿 4~5개)
- 군집: 5개 (업무/상업 혼합, 아침 도착 업무, 주거 도착, 생활·상권 혼합, 외곽 주거)
- 모델: LightGBM, 학습 목표는 오차 최소화(RMSE) 또는 건수형(Poisson)
- 핵심 피처: 시간대(출퇴근·야간·주말), 날씨, 입지(지하철·버스·표고), **날씨×시간대 상호작용**

#### 5) 핵심 결과 (표 1개, 4~5행)
| 구분 | 테스트 오차(RMSE) | 비고 |
|------|-------------------|------|
| 161개 최종 운영안 | 0.8604 | 정적 피처 + 날씨×시간대 상호작용 |
| 군집별 분기 적용 | 0.867~0.868 | 일반화 실패로 미채택 |
| 대표 15개 최선 | 0.799 (cluster02) | 탐색용 |

#### 6) 이미지 (1장에 1개)
- **후보**: 161개 운영 모델 비교 차트 (`ddri_analysis_ml_final_full161_test_rmse.png`)
- 또는 군집 프로파일 히트맵 (군집 특성 한눈에)

### 2.2 2장: 최종 모델 + 성능 + 결론

#### 1) 최종 채택안 (표 1개, 5~6행)
| 항목 | 내용 |
|------|------|
| 모델 | LightGBM |
| 데이터 | 강남구 전체 161개 |
| 피처 조합 | 정적 입지 + 날씨×시간대 상호작용 |
| 테스트 오차 | 0.8604 |
| 미채택 | 군집별 분기, 단순 가중치 |

#### 2) 핵심 판단 (불릿 4개)
- 군집화는 대여소 공간 역할 해석에 유효
- 대표 15개 실험은 군집별 피처 발굴에 유효
- 운영 기준은 **161개 단일 모델**이 더 안정적
- 최선 조합: 정적 피처 + 날씨×시간대 상호작용

#### 3) 결론 (2~3줄)
- 강남구 161개 대여소 시간대별 수요 예측, LightGBM 단일 모델로 운영
- 날씨만 단독이 아니라 **비·강수×출퇴근·야간** 상호작용이 핵심
- 다음 단계: 예측값 → 잔여 자전거·부족 위험 후처리 규칙 설계

#### 4) 이미지 (2장에 0~1개)
- **후보**: 15개 vs 161개 역할 구조 차트 (`ddri_analysis_ml_final_role_structure.png`)
- 또는 생략하고 표만으로 정리
