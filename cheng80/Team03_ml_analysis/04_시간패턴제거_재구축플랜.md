# 시간패턴 제거 재구축 플랜

작성일: 2026-03-16  
대상 입력: `3조 공유폴더/군집별 데이터_전체 스테이션/full_data/`  
대상 출력: `3조 공유폴더/군집별 데이터_전체 스테이션/bike_change_full161_base_outputs/`

## 1. 현재 확인 결과

### 1.1 기준 문서 해석

`cheng80/time_series_deseasonalization.md`의 핵심은 행 삭제가 아니라 아래 변환이다.

- `station_id + weekday + hour` 기준 평균 계산
- 원래 타깃에서 해당 평균을 빼서 계절성 제거

즉, 기본 방식은 **행 수를 줄이는 전처리**가 아니라 **타깃 값을 변환하는 전처리**다.

### 1.2 기존 산출물의 실제 행 감소 원인

기존 `bike_change_full161_base_outputs`는 `rental_count`가 아니라 `bike_change`를 만들었다.

- `bike_change_raw = rental_count.diff()` 사용
- 이 때문에 각 `station_id`의 첫 시점 1행이 `NaN`
- 결과적으로 분할마다 `161개 스테이션 x 1행 = 161행`만 제거됨

실제 삭제된 행:

- Train/Validation 원본(`2023+2024`): 각 스테이션의 `2023-01-01 00:00`
- Test 원본(`2025`): 각 스테이션의 `2025-01-01 00:00`

즉, 기존 출력에서 행이 거의 안 줄어든 이유는 정상이다.  
시간패턴 제거 자체는 행 삭제를 거의 만들지 않기 때문이다.

| 기존 train 출력 | 2,824,423 | 161 | 각 스테이션 첫 시점 diff 불가 |
| 기존 test 출력 | 1,410,199 | 161 | 각 스테이션 첫 시점 diff 불가 |

### 2.3 이번에 원하는 `rental_count` 시간패턴 제거 기준 예상 행 수

이번 작업을 템플릿 정책에 맞춰 `rental_count` 타깃으로 다시 하면:

- `seasonal_mean_2023 = mean(rental_count | station_id, weekday, hour)`를 `2023`만으로 계산
- `rental_count_deseasonalized = rental_count - seasonal_mean_2023`

이 경우에는 `diff()`를 쓰지 않으므로 **원칙적으로 행 삭제가 없다**.

예상 결과:

| 구분 | 시작 행 수 | 종료 행 수 | 비고 |
| --- | ---: | ---: | --- |
| Train+Validation 원본 | 2,824,584 | 2,824,584 | 삭제 없음 |
| Test 원본 | 1,410,360 | 1,410,360 | 삭제 없음 |

추가 확인:

- `2023`의 `station_id + weekday + hour` 조합 수는 `27,048`
- 이는 `161 x 7 x 24`와 동일
- 따라서 `2024`, `2025`에도 매핑 누락 없이 적용 가능

## 3. 기존 산출물의 문제점

기존 폴더가 이번 목적과 맞지 않는 이유는 아래와 같다.

- 타깃이 `rental_count`가 아니라 `bike_change`
- 시계열 템플릿의 학습 타깃과 불일치
- 근거 시각화가 히트맵 중심이라 "시간 패턴이 실제로 얼마나 제거됐는지"를 직관적으로 보기 어려움
- 노트북이 `bike_change` 기준이라 이번 실험 설계의 기록물로 재사용하기 부적절

## 4. 재구축 원칙

### 4.1 학습 정책 고정

- Train: `2023`
- Validation: `2024`
- Test: `2025`
- Validation 우세 모델 선택 후 `2023+2024` 재학습
- 최종 `2025` 평가

### 4.2 타깃 고정

- 원래 모델링 기준 타깃: `rental_count`
- 시간패턴 제거 타깃: `rental_count_deseasonalized`

### 4.3 시간패턴 제거 기준

1. `2023` 데이터만 사용해 seasonal table 생성
2. 키: `station_id + weekday + hour`
3. `seasonal_mean_2023` 계산
4. 모든 분할에 아래 식 적용

`rental_count_deseasonalized = rental_count - seasonal_mean_2023`

필요 시 비교용 보조 컬럼:

- `seasonal_mean_2023`
- `rental_count_deseasonalized`
- `residual_abs = abs(rental_count_deseasonalized)`

## 6. 차트 구성 원칙

히트맵만으로 끝내지 않는다. 아래 차트를 기본으로 둔다.

- 전체 평균 24시간 프로파일
  - 원본 vs 제거 후를 한 그래프에 중첩
  - `2023`, `2024`, `2025` 각각 저장
- 요일별 24시간 프로파일
  - 7개 요일 선그래프 또는 small multiples
- 대표 스테이션별 24시간 프로파일
  - 수요 상/중/하 3개 스테이션 이상
- 제거 전후 분포 비교
  - histogram 또는 KDE
- 패턴 강도 요약 막대그래프
  - 예: `weekday-hour mean absolute amplitude`

히트맵은 보조 자료로만 유지한다.
