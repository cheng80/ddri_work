## 2. ksm_note 대비


| 항목    | 템플릿                               | ksm_note                                                          | 일치 여부 |
| ----- | --------------------------------- | ----------------------------------------------------------------- | ----- |
| 시간 분할 | train=2023, valid=2024, test=2025 | train=2023, valid=2024, test=2025                                 | ✓ 동일  |
| 재학습   | 2023+2024 재학습 후 2025 평가           | `train_valid_df = concat(train, valid)` → 2023+2024 재학습 후 2025 평가 | ✓ 동일  |
| 데이터   | raw_data 공통 CSV                   | `second_round_feature_collection.csv`                             | ✗ 다름  |
| 타깃    | rental_count                      | rental_count                                                      | ✓ 동일  |
| 모델    | LinearRegression, LightGBM_RMSE   | LightGBM (2단계·소프트보팅은 구조 확장)                                       | △ 유사  |
| 평가    | RMSE, MAE, R²                     | RMSE, MAE, R², exact_match, bucket 등 추가                           | △ 확장  |


### 2.1 차이점 요약

- **학습 정책(시간 분할·재학습)**: 템플릿과 **동일**
- **데이터 소스**: 프로토콜은 "개인 가공 CSV를 정본처럼 쓰지 않는다"고 명시. ksm_note는 `second_round_feature_collection` 사용 → 정본이 아닌 2차 가공본 사용

## 3. hmw 대비


| 항목     | 템플릿                               | hmw                                                          | 일치 여부 |
| ------ | --------------------------------- | ------------------------------------------------------------ | ----- |
| 시간 분할  | train=2023, valid=2024, test=2025 | train=2023, valid=2024, test=2025                            | ✓ 유사  |
| 재학습    | 2023+2024 재학습 후 2025 평가           | `train_mask = year in [2023,2024]`, `test_mask = year==2025` | ✓ 동일  |
| 데이터    | raw_data (15개 대표)                 | `station_hour_bike_flow_*.csv` (161개 전체)                     | ✗ 다름  |
| 타깃     | rental_count                      | bike_change, bike_count_index                                | ✗ 다름  |
| 대여소 범위 | 군집별 3개 (총 15개)                    | 161개 전체                                                      | ✗ 다름  |
| 모델     | LinearRegression, LightGBM_RMSE   | hist_gbm_tuned, ridge 등                                      | ✗ 다름  |
| 평가     | RMSE, MAE, R²                     | RMSE, MAE, R²                                                | ✓ 동일  |


### 3.1 차이점 요약

- **학습 정책(시간 분할·재학습)**: 템플릿과 **유사**
- **실험 목적**: 템플릿은 `rental_count` 예측(대여건수). hmw는 `bike_change`(재고 흐름)·`bike_count_index` 예측 → **별도 실험 파이프라인**

## 4. 종합 요약


| 구분           | 학습 정책 일치 | 비고                                  |
| ------------ | -------- | ----------------------------------- |
| **ksm_note** | ✓ 동일     | 시간 분할·재학습 동일. 데이터만 second_round 사용. |
| **hmw**      | △ 유사     | 시간 분할·재학습 유사. 데이터·타깃·대여소 범위는 별도 실험. |


### 4.1 권장 사항

- **ksm_note**: 프로토콜 준수를 강화하려면 raw_data 기반 실험을 병행하거나, second_round 사용 근거를 문서화
- **hmw**: bike_change·station-hour 파이프라인은 템플릿과 목적이 다르므로, 별도 실험으로 명시하고 학습 정책만 참조용으로 정렬
